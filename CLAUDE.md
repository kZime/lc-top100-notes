# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Quick Start

Use `/lc` to set up any problem (run from `top100/`):

```
/lc 128
/lc longest-consecutive-sequence
```

After solving, run `/lc-review` from inside the problem folder to pull top LeetCode CN solutions, generate `notes.md`, and update status in README.

```bash
cd hot100/0128_Longest_Consecutive_Sequence
/lc-review
```

## Build & Run

Each problem has both a C++ and Go solution. Run them directly:

```bash
# C++
g++ -std=c++17 solution.cpp -o sol && ./sol

# Go
go run solution.go
```

The test harness in each file prints `✓` / `✗` per test case and exits non-zero on failure.

## Folder Structure

```
top100/
├── hot100/NNNN_Problem_Title/
│   ├── problem.md      ← problem statement
│   ├── solution.cpp    ← C++ solution + test harness
│   ├── solution.go     ← Go solution + test harness
│   └── notes.md        ← post-solve review (generated by /lc-review)
└── others/
    ├── contests/
    └── tmp/
```

Folder naming: zero-padded 4-digit ID + title words joined by `_` (e.g. `0049_Group_Anagrams`).

## README Status Legend

- `☑` — solved with optimal solution (confirmed by `/lc-review`)
- `⊙` — attempted but not yet optimal
- `☐` — not started

`/lc-review` only promotes a problem to `☑` if the user's solution matches the best known time/space complexity. The top progress counter (`**N / 100**`) reflects `☑` count only.

## Workflow Details

- `/lc` fetches problem data from LeetCode GraphQL (COM first, CN as fallback, built-in knowledge as last resort), creates the folder, writes `problem.md`, `solution.cpp`, `solution.go` with at least 5 test cases, and sets status to `⊙`.
- `/lc-review` reads all `solution*.cpp` / `solution*.go` files, fetches top CN solutions, distills 2–3 canonical algorithms ordered naive→optimal, maps the user's code to one of them, and generates `notes.md` in Chinese. If the user's solution is already optimal it marks the problem `☑` and removes it from the "尝试中" table.
- Multiple solution files (`solution2.cpp`, `solution3.cpp`, etc.) are all analyzed — append a new file to trigger incremental review without overwriting existing notes.
