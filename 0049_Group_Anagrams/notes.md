# 49. Group Anagrams — 题解笔记

## 核心算法

### 算法一：排序作为哈希键
**思路**：两个字符串互为字母异位词，当且仅当它们包含的字符完全相同。对每个字符串排序后得到一个规范形式（canonical key），互为异位词的字符串排序后必然相同。以排序后的字符串作为 `unordered_map` 的键，将所有原始字符串归入对应的桶中，遍历结束后输出所有桶即为结果。
**时间**：O(n · k log k)，n 为字符串数量，k 为单个字符串最大长度　**空间**：O(n · k)

### 算法二：字符计数作为哈希键
**思路**：同样利用"异位词字符组成相同"的性质，但避免排序开销。对每个字符串统计 26 个小写字母的出现次数，将该频次数组编码成固定格式的字符串（如 `"1#0#2#..."` 或 `"a1b0c2..."` ）作为哈希键。由于字母数固定为 26，构造键的时间为 O(k)，整体优于排序方案。
**时间**：O(n · k)　**空间**：O(n · k)

### 算法三：质数乘积作为哈希键（数学法）
**思路**：将 26 个字母各映射到一个不同的质数，对字符串中每个字符对应质数连乘。由唯一分解定理，两字符串若互为异位词则乘积相同，反之亦然。此方法构造键为 O(k)，且键为整数，哈希效率高；但字符串较长时乘积可能溢出 64 位整数，工程中较少使用。
**时间**：O(n · k)　**空间**：O(n)

## 我的解法

**使用算法**：算法一 — 排序作为哈希键

**判断依据**：
- 使用了 `unordered_map<string, vector<string>> mp`，以字符串为键
- 对每个字符串副本执行 `sort(c.begin(), c.end())`，将排序结果作为 map 的 key
- 将原始字符串 `ori_c` push 入对应桶，最终遍历 map 收集结果

以上三点与算法一的标准实现完全吻合。

**可改进点**：
- `for (auto c : strs)` 按值拷贝字符串，再 `string ori_c = c` 又做第二次拷贝，每次循环多一次不必要的分配。建议改为 `for (const string& s : strs) { string key = s; sort(key.begin(), key.end()); mp[key].push_back(s); }`，只拷贝一次，语义更清晰。
- 整体思路已是算法一的标准实现，时间和空间复杂度达到该算法的理论最优，无需进一步结构性优化。
