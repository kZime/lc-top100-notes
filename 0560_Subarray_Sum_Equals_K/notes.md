# 560. Subarray Sum Equals K — 题解笔记

## 题意

给定整数数组 `nums` 和整数 `k`，统计所有连续子数组中，元素之和恰好等于 `k` 的个数。数组可含负数，所以不满足单调性，不能用滑动窗口。

---

## 核心算法

### 算法一：前缀和 + 暴力枚举
**思路**：预先计算前缀和数组 `s[0..n]`，然后双重循环枚举所有子数组 `[i, j]`，通过 `s[j] - s[i-1] == k` 在 O(1) 时间判断是否符合条件。比起三重循环（边枚举边累加）少一层，但仍是平方复杂度。
**时间**：O(N²)　**空间**：O(N)

优化思路：每次枚举到 j，要向前循环去找有几个 s[i-1] 满足 s[j] - s[i-1] == k， 每找到一个就说明（i, j） 合法，这个过程可优化。

### 算法二：前缀和 + 哈希表（最优）
**思路**：核心等式 `s[j] - s[i] = k` 等价于 `s[i] = s[j] - k`。

从左到右遍历，用哈希表记录每个前缀和出现的次数。遍历到位置 `j` 时，直接查表得知有多少个历史前缀和等于 `s[j] - k`，即为以 `j` 结尾的满足条件的子数组数量。

初始化 `cnt[0] = 1` 是为了覆盖子数组从下标 0 开始的情况。
**关键洞察**：把"枚举左端点"这一步从 O(N) 用哈希表压缩到 O(1)，一次遍历完成。
**时间**：O(N)　**空间**：O(N)

## 我的解法

### solution.cpp

**使用算法**：算法一 — 前缀和 + 暴力枚举
**判断依据**：
- 代码显式构造了 `vector<int> s(n+10)` 前缀和数组，并用 `s[j] - s[i-1] == k` 判断
- 外层 `i` 枚举起点，内层 `j` 枚举终点，是标准 O(N²) 双循环

**可改进点**：
1. 可以升级为算法二（前缀和 + 哈希表），将时间复杂度从 O(N²) 降至 O(N)，在 n = 2×10⁴ 的数据规模下有显著提升
2. 数组 `s` 多分配了 10 个元素（`n+10`），实际只需 `n+1`
