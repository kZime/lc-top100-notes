# 155. Min Stack — 题解笔记

## 题意

设计一个支持 `push`、`pop`、`top` 和 `getMin` 的栈，并且要求这四个操作都尽量保持常数时间。
核心难点在于：普通栈只能直接拿到栈顶，无法在 `O(1)` 时间内知道当前最小值。

## 核心算法

### 算法一：普通栈 + 每次线性找最小值
**思路**：直接用一个栈保存所有元素，`push`、`pop`、`top` 都是常规操作。`getMin` 时再把栈内元素遍历一遍求最小值。这个方案最直观，但最小值查询会退化成线性时间，不满足题目要求。  
**时间**：`push/pop/top` 为 `O(1)`，`getMin` 为 `O(n)`　**空间**：`O(n)`

### 算法二：双栈同步记录前缀最小值
**思路**：一个主栈存真实数据，另一个辅助栈与主栈等长，记录“到当前位置为止的最小值”。每次 `push(val)` 时，辅助栈压入 `min(val, 当前最小值)`；每次 `pop()` 时两个栈同步弹出。这样辅助栈栈顶始终就是当前最小值。  
**时间**：`push/pop/top/getMin` 均为 `O(1)`　**空间**：`O(n)`

### 算法三：双栈 + 只记录新的最小值（最优）
**思路**：主栈照常存所有元素；辅助栈只在遇到“更小值”或“与当前最小值相等的值”时才入栈。这样既能在 `O(1)` 时间得到最小值，又能在弹出元素时，通过比较主栈栈顶和辅助栈栈顶决定是否同步弹出最小值。相比等长辅助栈，这种写法通常更省一些常数空间。  
**时间**：`push/pop/top/getMin` 均为 `O(1)`　**空间**：`O(n)`

### [solution.cpp](solution.cpp) — 算法三：双栈 + 只记录新的最小值
**判断依据**：代码里用了两个 `stack<int>`：`s` 作为主栈，`m` 作为最小值栈。`push` 时只有当 `m` 为空，或者 `val <= m.top()` 时才把元素压入 `m`；`pop` 时如果 `s.top() == m.top()` 就同步弹出 `m`。这正是“只记录当前最小值及其重复项”的经典双栈写法。
