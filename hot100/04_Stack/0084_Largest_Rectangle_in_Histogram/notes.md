# 84. Largest Rectangle in Histogram — 题解笔记

## 题意

给定一个柱状图，每根柱子的宽度都为 `1`，要求找出其中面积最大的矩形。
本质上是：把每根柱子都视作矩形的“短板高度”，求它向左右最多能扩展到哪里，然后据此计算最大面积。

## 核心算法

### 算法一：枚举每根柱子并向两侧扩展
**思路**：把每根柱子 `heights[i]` 当作候选矩形的高度，然后不断向左、向右扩展，直到遇到严格更矮的柱子为止。扩展区间的宽度乘上当前高度，就是以该柱子为短板时的面积。这个思路最直观，但每根柱子都可能重复扫描大量区间。  
**时间**：`O(n^2)`　**空间**：`O(1)`

### 算法二：单调栈预处理左右最近更小元素
**思路**：分别求出每根柱子左侧最近的严格更小元素位置、右侧最近的严格更小元素位置。这样以 `heights[i]` 为短板的最大宽度就是 `right[i] - left[i] - 1`，面积也就能直接算出来。求这两个数组时可以用单调递增栈在线性时间内完成。  
**时间**：`O(n)`　**空间**：`O(n)`

### 算法三：单调栈一次扫描出栈结算（最优）
**思路**：维护一个单调递增栈，栈里存柱子的下标。当当前柱子比栈顶更矮时，说明栈顶柱子右侧第一个更小值已经找到，可以立刻出栈并结算以它为短板的最大面积。通常会在数组两端补哨兵 `0`，把所有柱子的面积统一在一次扫描中算完，代码常数更小。  
**时间**：`O(n)`　**空间**：`O(n)`

## 我的解法

### [solution.cpp](solution.cpp) — 算法二：单调栈预处理左右最近更小元素
**判断依据**：代码先从左到右计算 `psi`（左侧最近更小元素下标），再从右到左计算 `nsi`（右侧最近更小元素下标），最后用 `heights[i] * (nsi[i] - psi[i] - 1)` 统计每根柱子作为短板时的最大面积。这正是“先求左右边界，再统一结算面积”的经典单调栈写法。  
**结论**：这份 C++ 解法的时间复杂度已经达到本题最优的 `O(n)`，算法层面没有问题。  
**可改进点(算法层面，和最佳实践建议，如果没有，省略)**：可以把“两次预处理 + 一次统计”进一步写成“单调栈一次扫描出栈结算”的形式，复杂度不变，但常数和代码长度通常会更好一些；另外测试代码里的 `DEBUG` 输出可以移除，避免提交到仓库时带入无关日志。
