# 394. Decode String — 题解笔记

## 题意

给定一个形如 `k[encoded_string]` 的编码字符串，要求把它还原成展开后的原始字符串。
难点在于括号可以嵌套，因此不能只做简单的线性替换，而要正确处理“进入一层括号、展开内部、再回到外层继续拼接”的过程。

## 核心算法

### 算法一：递归下降解析
**思路**：把整个字符串看成一段递归语法。遇到数字就解析重复次数，遇到 `[` 就递归处理子串，遇到 `]` 就返回当前层构造好的结果。递归天然适合处理嵌套结构，代码也比较贴合题意。  
**时间**：`O(n + m)`，其中 `n` 是输入长度，`m` 是最终输出长度　**空间**：`O(d + m)`，其中 `d` 是嵌套深度

### 算法二：双栈维护重复次数和历史字符串
**思路**：一个栈存每层括号前的重复次数，另一个栈存进入这一层之前已经构造好的字符串。扫描到 `[` 时把当前状态压栈并清空当前串；扫描到 `]` 时弹出栈顶，把当前串重复若干次后接回上一层。这样可以显式模拟递归展开过程。  
**时间**：`O(n + m)`　**空间**：`O(d + m)`

### 算法三：单栈保存上下文（最优）
**思路**：把“当前层的重复次数 + 当前层已构造字符串”合成一个上下文，压到同一个栈里。遇到字母就追加到栈顶字符串；遇到 `[` 就开启新上下文；遇到 `]` 就把当前层展开后并回上一层。和双栈本质相同，但状态更集中，写起来很顺手。  
**时间**：`O(n + m)`　**空间**：`O(d + m)`
