# 70. Climbing Stairs — 题解笔记

## 题意

爬 n 级台阶，每次可走 1 步或 2 步，求共有多少种走法。答案即斐波那契数列第 n 项（f(1)=1, f(2)=2）。

## 核心算法

### 算法一：动态规划（O(n) 空间）

**思路**：定义 `dp[i]` 为爬到第 i 级的方案数，状态转移 `dp[i] = dp[i-1] + dp[i-2]`，边界 `dp[1]=1, dp[2]=2`。线性遍历填表，结果为 `dp[n]`。

**时间**：O(n)　**空间**：O(n)

### 算法二：滚动数组 / 双变量 DP（最优）

**思路**：由于 `dp[i]` 只依赖前两项，可用两个变量 `a, b` 滚动更新，`(a, b) → (b, a+b)`，完全省去数组。O(n) 时间，O(1) 空间，是本题标准最优解。另一变体是用大小为 3 的滚动数组以 `i%3` 索引，本质等价但稍复杂。

**时间**：O(n)　**空间**：O(1)

## 我的解法

### [solution.cpp](solution.cpp) — 算法二：滚动数组 DP

**判断依据**：用 `array<int,3>` 以 `i%3` 为索引实现滚动，`dp[i%3] = dp[(i-1+3)%3] + dp[(i-2+3)%3]`，O(n) 时间 O(1) 空间，与算法二一致。

**可改进点**：用大小为 3 的环形数组略显冗余——因为每步只用到前两个值，直接两变量滚动（`a, b = b, a+b`）更简洁直观，且初始化逻辑更清晰（`dp[0]=1` 对应"到第 0 级的方案数为 1"需要额外思考）。复杂度相同，属于风格改进，不影响正确性。
