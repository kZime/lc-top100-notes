# 128. Longest Consecutive Sequence — 题解笔记

## 题意

给定一个无序整数数组，找出其中最长连续整数序列的长度（如 [1,2,3,4]），要求时间复杂度 O(n)，不能排序。

---

## 核心算法

### 算法一：哈希集合 + 序列起点判断

**思路**：将所有数放入哈希集合，然后只从"序列起点"（即 x-1 不在集合中的数）开始向右枚举，累计连续长度。关键优化：非起点的数跳过不处理，保证每个数至多被访问两次（一次判断是否起点，一次在某个起点的扩展中被访问）。遍历集合而非原数组，避免重复元素导致 O(n²)。

**时间**：O(n)　**空间**：O(n)

**关键洞察**：从非起点出发的序列一定是某起点序列的子集，跳过这些非起点是 O(n) 的本质。
跳过所有有前驱的数 （if (vis.count(x-1) continue)）

---

### 算法二：哈希表记录链长度（区间合并）

**思路**：用 `len[x]` 表示包含 x 的连续序列的长度。处理每个新数 x 时，查找左邻 `len[x-1]` 和右邻 `len[x+1]` 的链长，将三段合并：新链长 = left + 1 + right。同时更新链两端端点的 len 值（只需更新最左端和最右端，中间值不影响结果）。

**时间**：O(n)　**空间**：O(n)

**适用场景**：如果需要动态插入数字并实时查询最长序列，此方案扩展性更好。

---

### 算法三：并查集（Union-Find）

**思路**：将每个数映射到数组索引，建立并查集。对每个数，向左右扩展查找相邻数，若存在则将两者合并入同一集合，并标记已合并的邻居（避免重复合并）。最终各集合的 size 就是对应连续序列的长度，取最大值。

**时间**：O(n α(n)) ≈ O(n)　**空间**：O(n)

---

## 我的解法

**使用算法**：算法三 — 并查集

**判断依据**：
- 代码中定义了 `struct unionSet`，含 `f`（父节点）、`sz`（集合大小）
- `get_root` 使用路径压缩：`f[x] = get_root(f[x])`
- 用 `vis` 哈希表映射值→索引，通过 `vis[b] = -1` 标记已合并的邻居
- 外层循环对每个元素双向扩展，调用 `un.merge(i, vis[b])` 合并相邻节点

**可改进点**：

1. **`merge` 未按 size 合并（主要问题）**：当前 `merge` 的逻辑是将 index 较小的 root 合并到 index 较大的 root（`if (y > x) swap(x, y)` 是基于数组下标，不是基于树的大小）。这不是真正的"按秩合并"，单靠路径压缩理论上是 O(n log n)。应改为按 `sz` 大小合并：

   ```cpp
   void merge(int x, int y) {
       int fx = get_root(x), fy = get_root(y);
       if (fx == fy) return;
       if (sz[fx] < sz[fy]) swap(fx, fy);
       sz[fx] += sz[fy];
       sz[fy] = 0;
       f[fy] = fx;
   }
   ```

2. **`!vis.count(a)` 是死代码**：`vis` 是从 `nums` 构建的，`a = nums[i]`，所以 `vis.count(a)` 恒为 1，该条件永不触发，可删除。

## 我的解法（solution2.cpp）

**使用算法**：算法一 — 哈希集合 + 序列起点判断（erase 替代标记）

**判断依据**：
- 使用 `unordered_set` 存储所有数，O(1) 查找
- 两行条件分别处理两种跳过情形：`!vis.count(a)` 跳过已 erase 的元素；`vis.count(a-1)` 跳过非起点（左邻存在时说明不是序列最左端）
- 向右单向扩展 (`b = a+1; while vis.count(b)`)，边扩展边 `erase`，保证每个元素至多被访问两次
- 全部测试用例通过（8/8）

**可改进点**：已是算法一的简洁实现，无明显改进点。与 solution.cpp（并查集）相比，代码更短、常数更小，是本题更推荐的写法。
