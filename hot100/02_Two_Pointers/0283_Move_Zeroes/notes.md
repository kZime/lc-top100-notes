# 283. Move Zeroes — 题解笔记

## 题意

给定整数数组，原地把所有 0 移到末尾，同时保持非零元素的相对顺序不变，不得使用额外数组。

## 核心算法

### 算法一：两次遍历（非零前移 + 尾部补零）
**思路**：第一次遍历用指针 `j` 收集所有非零元素并写到数组前端（`nums[j++] = nums[i]`）；第一次结束时 `j` 就是非零元素的个数，第二次从 `j` 开始把剩余位置全部赋 0。逻辑清晰，但需要两趟扫描。  
**时间**：O(n)　**空间**：O(1)

### 算法二：单次遍历双指针交换（最优）
**思路**：`left` 指向下一个待填的位置，`right` 逐个向右扫描。每当 `right` 遇到非零元素，就把它与 `nums[left]` 互换，然后 `left++`。由于 `left` 左边始终是已处理的非零区，`left` 到 `right-1` 之间始终全是 0，每次交换都是把一个 0 换到 `right` 位置、把非零换到前面，且非零的相对顺序不受影响。一趟完成，常数更小。  
**时间**：O(n)　**空间**：O(1)

## 我的解法

### [solution.cpp](solution.cpp) — 算法二：双指针交换

**判断依据**：使用两个指针 `l`（指向 0）和 `r`（指向非零）交替推进并执行 `swap`，与算法二本质相同；时间 O(n)、空间 O(1)，与最优方案一致。

**可改进点**：实现比标准写法繁琐。前两个单独的初始化循环（找第一个 0、找第一个非零）完全可以省去——`left=0` 直接出发，只需一个 `right` 外层循环：

```cpp
int left = 0;
for (int right = 0; right < n; right++) {
    if (nums[right] != 0) {
        swap(nums[left], nums[right]);
        left++;
    }
}
```

当前写法有三个循环、多个边界判断，代码量约为标准写法的 3 倍，可读性较低，建议重写为上述形式以练熟模板。
