# 15. 3Sum — 题解笔记

## 题意

给定整数数组，找出所有和为 0 的不重复三元组。核心难点不在于找三元组，而在于**去重**——元素可以重复，但输出中不能有重复的三元组。

---

## 核心算法

### 算法一：暴力三重循环
**思路**：三层嵌套枚举所有三元组，找到和为 0 的则规范化为 (min, mid, max) 后加入结果，最后 sort + unique 去重。
**时间**：O(N³)　**空间**：O(K)（K 为结果数量）
**注**：在 N=3000 时会 TLE，仅作思路验证用。

### 算法二：哈希集合 + 分类讨论
**思路**：统计每个值的出现次数，对去重后的值集合分三类讨论：① 三个相同值（只有 0,0,0 满足）；② 两个相同值 `a,a,-2a`（需 `cnt[a]≥2` 且 `-2a` 存在）；③ 三个不同值 `a<b<c`，枚举有序对 (a,b)，验证 `c=-a-b` 是否存在于集合中且 `c>b`。由于在有序 set 上枚举 `a<b` 并限定 `c>b`，天然无重复。
**时间**：O(N + U²)，U 为不同值数量，最坏 O(N²)　**空间**：O(U)

### 算法三：排序 + 双指针
**思路**：先排序。外层枚举第一个元素 `nums[i]`（固定最小值），内层用双指针 `L=i+1, R=n-1` 向中间夹逼。若三数之和 < 0，L 右移；> 0，R 左移；= 0，记录后同时内缩。去重只需跳过相邻重复值：外层 `nums[i]==nums[i-1]` 跳过，内层找到答案后跳过重复的 `nums[L]` 和 `nums[R]`。排序保证了不会输出重复三元组。
**时间**：O(N log N + N²) = O(N²)　**空间**：O(1)（排除输出）
**关键洞察**：排序是去重的前提，双指针把内层两重循环压缩到 O(N)。

## 我的解法

### [solution.cpp](solution.cpp) — 算法一：暴力三重循环
**判断依据**：三重嵌套循环，每找到一个三元组就用 `min/max/-a-c` 规范化，最后 `sort + unique` 去重，是暴力枚举的标准写法。
**可改进点**：复杂度 O(N³)，提交会 TLE。升级路径是算法三（排序+双指针），代码量相近但效率从 O(N³) 降到 O(N²)。

### [solution2.cpp](solution2.cpp) — 算法二：哈希集合 + 分类讨论
**判断依据**：构建 `set<int>`（排序去重）+ `unordered_map`（计数），按三种情况枚举：全相等 / 两个相等 / 全不等，天然保证输出无重复。
**可改进点**：`vis`（`unordered_set`）和 `cnt` 存储相同元素，`vis` 仅为 O(1) 查找而设，可直接用 `cnt.count(c)` 替代，消除冗余结构。整体思路正确，复杂度 O(N²)。

### [solution3.cpp](solution3.cpp) — 算法三：排序 + 双指针
**判断依据**：先 `sort`，外层枚举 `nums[i]` 并跳过重复（`nums[i]==nums[i-1]`），内层双指针 `l=i+1, r=n-1` 夹逼，命中后 `l++` 并跳过重复的 `nums[l]`，是该算法的标准实现。
**可改进点**：找到答案后只跳过了左侧重复（`l` 侧），未显式跳过右侧重复（`r` 侧）。虽然 `l<r` 的终止条件最终仍会正确停止，但补上 `while (nums[r]==nums[r+1] && l<r) r--` 可使逻辑更对称、更易读。其余已是最优实现。
