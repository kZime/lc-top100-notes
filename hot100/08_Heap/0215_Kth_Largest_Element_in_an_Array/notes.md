# 215. Kth Largest Element in an Array — 题解笔记

## 题意

给定整数数组 `nums` 和整数 `k`，返回数组中第 k 大的元素（按排序顺序，非去重后的第 k 大）。

## 核心算法

### 算法一：排序
**思路**：直接对数组排序，返回下标 `n - k` 处的元素。思路最直接，代码一行，但没有利用"只需要找第 k 大"这一信息，做了多余的工作。
**时间**：O(n log n)　**空间**：O(1)（原地排序）

### 算法二：最小堆（大小为 k）
**思路**：维护一个大小为 k 的最小堆。遍历数组，若当前元素大于堆顶则弹出堆顶并将当前元素入堆，保证堆中始终是迄今见过的最大 k 个数。遍历结束后堆顶即为第 k 大。
**时间**：O(n log k)　**空间**：O(k)
**关键洞察**：`k` 远小于 `n` 时（如 Top-K 流式场景），堆方案优于排序；且堆不要求数据全部到达后才开始处理。

### 算法三：快速选择（随机化）
**思路**：基于快排的 partition：随机选 pivot，将数组划分为 `< pivot`、`= pivot`、`> pivot` 三段（三路划分可避免重复元素退化）。若 pivot 落在目标位置则直接返回，否则只递归一侧，每轮问题规模平均减半。随机化选 pivot 使期望复杂度为 O(n)。
**时间**：期望 O(n)，最坏 O(n²)（随机化后极难触发）　**空间**：O(log n)（递归栈）
**关键洞察**：这是面试最常考的"标准答案"。不用三路划分时遇到大量重复元素会退化；随机化解决了"精心构造反例"的问题。

## 我的解法

### [solution.cpp](solution.cpp) — BFPRT（中位数的中位数，确定性 O(n)）

**判断依据**：
- 将数组按每组 5 个切分，对每组**插入排序（降序）**，取组内中位数下标 `(l+r)/2` 处的元素，收集到 `meds` 数组。
- 递归调用自身求 `meds` 的第 `⌈meds.size()/2⌉` 大元素，即"中位数的中位数（MOM）"作为 pivot。
- 以 MOM 做**三路划分**（`lft > mom`，`rgt < mom`，`equ = mom`），根据 k 与 `lft.size()`、`equ` 的关系决定向哪侧递归。
- 这是 BFPRT 算法的完整实现，**最坏情况也是 O(n)**，理论上比随机化快速选择更强。

**可改进点**：
1. **残留调试输出**（第 19–23 行、第 55 行的 `cout`）：提交 LeetCode 时会产生多余输出，应删除。`solution1.cpp` 已清理。
2. **`nums.clear()` 时机**：三路划分结束后执行 `nums.clear()`，语义上略显奇怪——此时 `lft`/`rgt`/`equ` 已计算完毕，`nums` 内容已不再需要，clear 无害但多余；可直接删掉这行。

### [solution2.cpp](solution2.cpp) — 随机化快速选择（两路划分）

**判断依据**：
- 将"第 k 大"转化为升序排列下下标 `target = n - k` 处的元素。
- 随机选 pivot 后进行两路划分：`<= pivot` 的元素交换到左侧，`> pivot` 的保留右侧。pivot 归位后比较 `target` 与 pivot 下标，向对应一侧递归。
- 对应**算法三（随机化快速选择）的简化版**，但使用了两路（非三路）划分。

**可改进点**：
1. **两路划分在大量重复元素时退化**：当数组中有大量等于 pivot 的元素时，每轮划分极度不均（所有等于 pivot 的元素全部划入同一侧），导致递归深度 O(n)，整体退化为 O(n²)。作者注释已标注 `// TLE at many same elements`，确认了此问题。使用三路划分（见 `solution3.cpp`）可彻底解决。

### [solution3.cpp](solution3.cpp) — 随机化快速选择（三路划分）

**判断依据**：
- 同样将"第 k 大"转化为下标 `target = n - k` 处的元素。
- 随机选 pivot 后执行**荷兰国旗三路划分**：`lit`/`i`/`rit` 将数组切为 `< pivot`、`== pivot`、`> pivot` 三段（`[l, lit)` 全小于，`[lit, rit]` 全等于，`(rit, r]` 全大于）。 
- 若 `lit <= target <= rit` 则 `target` 落在等于区，直接返回 `pivot`；否则向小于区或大于区递归。
- 这是**算法三（随机化快速选择 + 三路划分）的完整标准实现**，是面试中该类题的"标准答案"。

**可改进点**：
- 已是随机化快速选择的最优写法，无明显改进点。
- 与 `solution1.cpp`（BFPRT）相比：两者期望复杂度均为 O(n)，但 BFPRT 有**最坏 O(n) 保证**，而本方案最坏 O(n²)（随机化后概率极低）；在实际 LeetCode 随机数据上本方案常数更小、跑得更快。
