# 215. Kth Largest Element in an Array — 题解笔记

## 题意

给定整数数组 `nums` 和整数 `k`，返回数组中第 k 大的元素（按排序顺序，非去重后的第 k 大）。

## 核心算法

### 算法一：排序
**思路**：直接对数组排序，返回下标 `n - k` 处的元素。思路最直接，代码一行，但没有利用"只需要找第 k 大"这一信息，做了多余的工作。
**时间**：O(n log n)　**空间**：O(1)（原地排序）

### 算法二：最小堆（大小为 k）
**思路**：维护一个大小为 k 的最小堆。遍历数组，若当前元素大于堆顶则弹出堆顶并将当前元素入堆，保证堆中始终是迄今见过的最大 k 个数。遍历结束后堆顶即为第 k 大。
**时间**：O(n log k)　**空间**：O(k)
**关键洞察**：`k` 远小于 `n` 时（如 Top-K 流式场景），堆方案优于排序；且堆不要求数据全部到达后才开始处理。

### 算法三：快速选择（随机化）
**思路**：基于快排的 partition：随机选 pivot，将数组划分为 `< pivot`、`= pivot`、`> pivot` 三段（三路划分可避免重复元素退化）。若 pivot 落在目标位置则直接返回，否则只递归一侧，每轮问题规模平均减半。随机化选 pivot 使期望复杂度为 O(n)。
**时间**：期望 O(n)，最坏 O(n²)（随机化后极难触发）　**空间**：O(log n)（递归栈）
**关键洞察**：这是面试最常考的"标准答案"。不用三路划分时遇到大量重复元素会退化；随机化解决了"精心构造反例"的问题。

## 我的解法

### [solution.cpp](solution.cpp) — BFPRT（中位数的中位数，确定性 O(n)）

**判断依据**：
- 将数组按每组 5 个切分，对每组**插入排序（降序）**，取组内中位数下标 `(l+r)/2` 处的元素，收集到 `meds` 数组。
- 递归调用自身求 `meds` 的第 `⌈meds.size()/2⌉` 大元素，即"中位数的中位数（MOM）"作为 pivot。
- 以 MOM 做**三路划分**（`lft > mom`，`rgt < mom`，`equ = mom`），根据 k 与 `lft.size()`、`equ` 的关系决定向哪侧递归。
- 这是 BFPRT 算法的完整实现，**最坏情况也是 O(n)**，理论上比随机化快速选择更强。
