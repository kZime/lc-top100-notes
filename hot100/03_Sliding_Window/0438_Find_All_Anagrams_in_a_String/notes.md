# 438. 找到字符串中所有字母异位词 — 题解笔记

## 题意

给定字符串 s 和 p，返回 s 中所有 p 的异位词（字母相同但顺序任意）的起始下标。由于异位词长度固定为 |p|，本质是大小固定的滑动窗口匹配问题。

## 核心算法

### 算法一：暴力 + 排序比较
**思路**：枚举 s 中所有长度为 |p| 的子串，排序后与排序后的 p 比较，相等则记录起始下标。
**时间**：O(n · m log m)（m = |p|）　**空间**：O(m)

### 算法二：定长滑动窗口 + 频率数组比较
**思路**：预处理 p 的字符频率数组 pCnt[26]，维护当前窗口的 sCnt[26]；每次滑动时加入新字符、移出最左字符，然后比较两个频率数组是否相等（26 次比较，O(1)）。
**时间**：O(n + m)　**空间**：O(|Σ|) = O(1)

### 算法三：定长滑动窗口 + notFit 计数器（最优）
**思路**：在算法二基础上维护整数 `notFit`，记录仍未满足的字符种数（pCnt[c] 从正变0时 notFit--，从0变正时 notFit++）。当 notFit == 0 时窗口是异位词，无需逐位比较频率数组，O(1) 判断。
**时间**：O(n + m)　**空间**：O(|Σ|) = O(1)

## 我的解法

### [solution.cpp](solution.cpp) — 算法三：定长滑动窗口 + notFit 计数器
**判断依据**：初始化 `pCnt` map 和 `notFit = pCnt.size()`，先处理前 k 个字符建立初始窗口，再滑动：移出 s[j-k] 时若 pCnt[s[j-k]] 从0变1则 notFit++，加入 s[j] 时若 pCnt[s[j]] 从1变0则 notFit--，notFit==0 时记录答案——与算法三完全吻合。复杂度 O(n+m) / O(1)，已达最优。
**可改进点**：可用 `int pCnt[26]` 替代 `unordered_map<char,int>`（题目保证均为小写字母），避免哈希开销，常数更小，但渐近复杂度不变。
