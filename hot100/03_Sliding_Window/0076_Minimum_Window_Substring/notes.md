# 76. 最小覆盖子串 — 题解笔记

## 题意

给定字符串 s 和模式串 t，找 s 中包含 t 所有字符（含重复）的最短子串。要求返回子串本身，不存在则返回空串。

## 核心算法

### 算法一：暴力枚举
**思路**：枚举 s 的所有子串，对每个子串统计字符频率后与 t 比较，取满足条件的最短者。
**时间**：O(n² · |Σ|)　**空间**：O(|Σ|)

### 算法二：滑动窗口 + 差值 Map（用户方法）
**思路**：用差值 map `tCnt` 记录各字符的"仍需数量"（正值=仍需，负值=已超量，0=恰好满足）。r 右移加字符（tCnt[s[r]]--），通过 check lambda 遍历 map 判断是否合法；找到合法窗口后，l 在满足 `tCnt[s[l]]+1 ≤ 0`（有余量）时右移收缩，持续更新最短窗口。
**时间**：O(n · |Σ_t|)（check 每次遍历 t 的唯一字符数，≤52 视为常数，故实际 O(n)）　**空间**：O(|Σ|)

### 算法三：滑动窗口 + valid 计数器（最优）
**思路**：在差值 map 基础上额外维护整数 `valid`，记录已完全满足的字符种数（tCnt[c] 从正变0时 valid++，从0变负时不变；l 收缩时反向）。当 valid == t 唯一字符数时窗口合法，O(1) 判断，整个主循环合并初始扩展和收缩，无需独立 check lambda，代码更简洁。
**时间**：O(n + m)　**空间**：O(|Σ|)

## 我的解法

### [solution.cpp](solution.cpp) — 算法二：滑动窗口 + 差值 Map
**判断依据**：使用 `tCnt` 差值 map + check lambda，先找初始合法窗口，再进入主循环滑动，收缩条件 `tCnt[s[l]]+1 ≤ 0` 与算法二完全一致。
**可改进点**：
1. 用整数 `valid` 取代 check lambda，可将初始找窗口与主循环合并，结构更清晰，O(1) 判断合法性。
2. 主循环的 `tCnt[s[l]]` 会为不在 t 中的字符在 map 中创建值为 0 的条目（operator[] 副作用），对不在 t 中的 s[l] 这些条目会被反复创建，略浪费内存（不影响正确性）。

---

### [solution2.cpp](solution2.cpp) — 算法三：滑动窗口 + unValid 计数器
**判断依据**：引入整数 `unValid`（记录尚未满足的字符种数），初始化阶段统计 t 中不同字符数，right 扩张时 `tCnt[s[r]]--`，归零时 `unValid--`；left 收缩条件 `tCnt[s[l]]+1 ≤ 0` 确保只移除有余量的字符，不会破坏合法性。整体 O(n)，与算法三一致。两段结构（先找初始窗口 + 主循环）与标准单循环等价，仅风格差异。
