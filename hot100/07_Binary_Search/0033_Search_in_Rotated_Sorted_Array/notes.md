# 33. Search in Rotated Sorted Array — 题解笔记

## 题意

给定一个**升序、无重复、可能经过一次旋转**的整数数组和一个目标值，要求以 **O(log n)** 时间找到目标值的下标，不存在则返回 -1。核心难点：旋转打破了全局有序性，需要在"局部有序"的结构上做二分。

---

## 核心算法

### 算法一：两次二分（先找旋转点）
**思路**：第一次二分找到旋转点（最小值下标 `i`），由此将数组分为两段递增子数组 `[0, i-1]` 和 `[i, n-1]`。再根据 `target` 与 `nums[n-1]` 的大小关系确定 target 在哪一段，对该段做普通二分查找。
**时间**：O(log n)　**空间**：O(1)
**适用**：思路清晰，分段明确，容易理解和复现。

### 算法二：一次二分（判断有序半边）
**思路**：每次取 `mid`，通过比较 `nums[l]` 与 `nums[mid]` 判断**哪半边是有序的**。有序的那半边可以用普通的大小判断确定 target 是否在其中，从而收缩边界。整个过程不需要预先找旋转点，一趟扫描完成。
**时间**：O(log n)　**空间**：O(1)
**关键洞察**：旋转数组每次从中间切开，至少有一半是连续有序的，利用这个性质可以每步缩小一半搜索空间。

### 算法三：XOR 压缩（极简写法）
**思路**：将三个布尔条件（`nums[0] > target`、`nums[0] > nums[mid]`、`target > nums[mid]`）用 XOR 组合成单一条件来决定 `lo = mid + 1` 还是 `hi = mid`。本质是算法二的逻辑压缩，代码量极小但可读性较低。
**时间**：O(log n)　**空间**：O(1)

## 我的解法

### [solution.cpp](solution.cpp) — 算法一变体：两次二分（找最大值下标）

**判断依据**：代码先用二分寻找局部最大值位置 `l`（`nums[mid] > nums[l]` 时扩大 `l`），以此作为旋转点。找到后再用 `lower_bound` 在左半段或右半段查找 target。这与标准"找最小值"方向相反但原理一致，属于两次二分方案。

**可改进点**：
1. **`size ≤ 3` 分支冗余**：二分逻辑在 `l+1 < r` 条件下能正确处理 size=2、3 的数组，无需单独线性扫描。删掉这段代码后所有测试仍通过。
2. **可简化为一次二分（算法二）**：直接在同一个 while 循环里判断有序半边并缩范围，减少一轮二分和分段 `lower_bound` 调用，代码更紧凑。

### [solution1.cpp](solution1.cpp) — 算法一：两次二分（找最小值下标）

**判断依据**：第一趟 `while(l < r)` 通过比较 `nums[mid]` 与 `nums[r]` 找到 `minIdx`（右段起点 / 最小值下标）；第二趟根据 `target >= fst` 判断所在段，调用 `lower_bound` 完成查找。属于两次二分方案的标准实现。

**可改进点**：无明显 bug，逻辑清晰。`while(l < r)` 需要搭配"找最小值"的比较策略（`> nums[r]` → 右移），稍不直觉；若改用 `while(l <= r)` 配合 `mid±1` 收缩，可与 solution2.cpp 写法统一风格。

---

### [solution2.cpp](solution2.cpp) — 算法二：一次二分（判断有序半边）

**判断依据**：单趟 `while(l < r)`，每次通过 `nums[l] <= nums[mid]` 判断左半边是否有序，再用范围判断决定向哪侧收缩。不需要预先找旋转点，一趟完成。循环退出时 `l == r`，补一行 `nums[l] == target ? l : -1` 处理最后一个元素。

**可改进点**：已是该问题的最优实现，O(log n) 时间、O(1) 空间，无明显改进点。`while(l < r)` + 循环后补检查是一种有效写法，等价于 `while(l <= r)` 版本。
