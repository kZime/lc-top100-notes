# 189. Rotate Array — 题解笔记

## 题意

给定整数数组 `nums`，将其**向右旋转 k 步**（原地修改）。本质上就是把数组末尾的 k 个元素搬到最前面，剩余元素依次后移。

---

## 核心算法

### 算法一：额外数组法
**思路**：新建长度为 n 的数组，将原数组下标 i 的元素放到新数组下标 `(i+k)%n` 的位置，最后拷贝回去。一次遍历直接定位，思路最直观。
**时间**：O(n)　**空间**：O(n)

### 算法二：三次反转法
**思路**：先将整个数组翻转，末尾 k 个元素就跑到了最前面，但顺序是反的；再分别翻转 `[0, k-1]` 和 `[k, n-1]` 两段，即可复原各自的顺序。三次 reverse 合计操作 2n 次，空间仅用双指针。
**时间**：O(n)　**空间**：O(1)
**关键洞察**：翻转 = 镜像，两次镜像 = 原序，这是处理"循环移位"类问题的经典技巧。

### 算法三：环状替换法
**思路**：将每个元素直接放到它旋转后的目标位置，用临时变量保存被覆盖的值。每条"旋转链"的长度为 `lcm(n,k)/k`，共有 `gcd(n,k)` 条链，每个元素恰好被访问一次。
**时间**：O(n)　**空间**：O(1)
**适用场景**：追求 O(1) 空间且不想写 reverse 辅助函数时的备选。

---

## 我的解法

**使用算法**：算法一 — 额外数组法（变体写法）
**判断依据**：代码分配了 `vector<int> tmp`，先用循环把后 k 个元素（`nums[n-k]` 到 `nums[n-1]`）push 进去，再追加前 n-k 个元素，最后整体赋回 nums。这正是"先收集旋转后的前半段，再拼后半段"的直接拼接方式，与标准额外数组法 `newArr[(i+k)%n] = nums[i]` 等价。

**可改进点**：
1. **编译 Bug**：类名写的是 `Solution_bf`，但测试 harness 里用的是 `Solution sol`，会导致编译报错（`Solution` 未定义）。需将类名改为 `Solution`，或在 harness 里对应修改。
2. **空间优化**：如希望 O(1) 空间，改用三次反转法（先 reverse 全体，再 reverse 前 k 个，再 reverse 后 n-k 个），逻辑同样简洁，且无需额外 vector。
